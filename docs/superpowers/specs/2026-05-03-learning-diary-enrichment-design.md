# 학습 일기 콘텐츠 풍부화 설계

**날짜**: 2026-05-03  
**대상**: `src/data/ld_p1.ts` ~ `src/data/ld_p10.ts` (100개 엔트리)

---

## 목표

현재 2~4문장으로 이루어진 학습 일기 본문을 8~10문장의 자연스러운 일기체로 확장한다. 날씨·감정·배경을 추가해 실제 일기처럼 보이도록 한다.

---

## 확장 기준

| 항목 | 현재 | 변경 후 |
|------|------|---------|
| 본문 문장 수 | 2~4문장 | 8~10문장 |
| 날씨 묘사 | 없음 | 자연스러운 위치에 1문장 이상 |
| 어휘 (`vocabulary`) | 3개 | 4~5개 |
| 문법 포인트 (`grammarPoints`) | 2개 | 2~3개 |
| 퀴즈 (`quiz`) | 3개 | 3개 (고정) |

---

## 본문 확장 패턴

카테고리·분위기에 맞게 자연스럽게 변형하되, 아래 흐름을 기본으로 한다:

1. **도입** — 날짜 감각, 요일, 계절 힌트
2. **날씨/배경** — 자연스러운 위치에 삽입 (강제 첫 문장 아님)
3. **사건 전개** — 구체적 묘사 (who, what, where)
4. **감정/반응** — 느낌, 감상
5. **마무리** — 한 줄 총평 또는 다음을 기대하는 문장

---

## 레벨별 문법 복잡도

- **초급**: `〜ました` / `〜です` / `〜でした` 중심, 단문 위주
- **중급**: `〜てから` / `〜ので` / `〜ている` / `〜たい` 포함, 2절 복문 가능
- **고급**: `〜ようになる` / `〜てしまう` / `〜ばよかった` / `〜と思う` 등 복합 표현

---

## 데이터 형식

`contentJp: RubySegment[]` 형식 유지. 한자에는 `ruby` 필드 필수, 히라가나·카타카나는 `text`만.

`contentKo`는 `contentJp`의 완전한 한국어 번역.
